### 2.4 Керування станами процесів

2.4.1 У поточному (1-му) псевдотерміналі виконати команду ping localhost у
фоновому режимі.

![image](https://github.com/Sergiy-Pats/ttt/assets/78663930/c481b375-c832-4dec-bea4-0a35789878a6)

2.4.2 Запустити 2-й псевдотермінал роботи із віртуальною ОС.

![image](https://github.com/Sergiy-Pats/ttt/assets/78663930/ed4f52b6-634b-4864-8428-d801a1ee3977)

2.4.3 У 2-му псевдотерміналі:
- для команди ping отримати таблицю її процесу (стовпчики PID, STAT, CMD);
- призупинити виконання процесу команди ping;
- переглянути процес призупиненої команди, враховуючи background-режим;
- отримати таблицю процесу команди ping;
- відновити виконання призупиненого процесу ping.
- остаточно завершити роботу процесу команди ping.

![image](https://github.com/Sergiy-Pats/ttt/assets/78663930/7fb69281-4431-4279-9e50-1e63009f8c75)

without hanging up».
У другому псевдотерміналі для команди ping отримати таблицю її процесу
(стовпчики PID, PPID, TTY, STAT, CMD) для вашого поточного користувача.

PID, PPID, TTY, STAT, CMD) для вашого поточного користувача.
Порівняти зміст таблиці процесу з пунктом 2.4.4, зробивши висновок про стан

### 2.5 Керування пріоритетами виконання процесів

![image](https://github.com/Sergiy-Pats/ttt/assets/78663930/87f43b85-2865-4a06-ab54-6c89e1a12d9d)

у фоновому режимі, враховуючи наступне:
- команду необхідно виконати від імені користувача root;
- команда повинна посилати мережеві запити кожні 0.0001*n секунди, де n – номер
вашої варіанту;
